## 소개
**DOKKAEBI-INVENTORY**는 드론 동아리/팀 운영을 위한 **재고 관리 + 입·출고 기록** 콘솔이야.  
브라우저에서 동작하는 **정적 웹앱**이고, **Firebase RTDB**랑 연동해서 여러 명이 동시에 접속해도 실시간 동기화돼.

- 배포: **GitHub Pages** (`/#/` 해시 라우팅)
- 저장: **LocalStorage + Firebase RTDB** 동시 사용
- 버전: 헤더에 `REACT_APP_VERSION` 노출, 필요 시 로컬 라벨 더블클릭으로 수정 가능

---

## 핵심 기능
- **재고 관리 (CRUD)**  
  - 장소/카테고리/품목 단위로 관리  
  - 검색 및 상세 팝업 지원
- **입·출고 기록 페이지 (`/#/logs`)**  
  - 필터(날짜/장소/품목), CSV/Excel 내보내기  
  - 동일 키(장소+품목+입/출) 1시간 이내는 자동 병합
- **검색 → 위치 이동**  
  - 결과 클릭 시 해당 품목 위치로 스크롤, 필요한 `details` 자동 오픈
- **반응형 UI**  
  - 720–1099px: 카드 2×2  
  - ≥1100px: 3열 + “전체” 카드 가운데 고정  
  - 모바일: 장소 카드 세로 스크롤 유지, 헤더 우측 로그아웃 버튼 보장
- **상세 팝업 UX**  
  - 팝업 열릴 때 내부 `details`를 모두 펼쳐서 보기
- **관리자/로그인**  
  - 로그인 화면 → 학번 + 이름 + 비밀번호(`2500`) 입력  
  - 관리자 모드에서만 수정/삭제/추가 버튼 노출  
  - 10분 무활동 시 자동 로그아웃
- **📤 재고 Excel 내보내기**  
  - 세 장소 모두 1행씩 배출(누락 없음)  
  - 수량 0은 숫자 `0`이 아니라 문자열 `"nan"`으로 기록 (업로더 무시 처리용)  
  - 시트2에 품목별 합계 (숫자)
- **📥 일괄 추가(베타)**  
  - `.xlsx`/`.csv`(첫 시트) 업로드  
  - **합산 모드**: 기존 수량에 가산  
  - **초기화 후 적용**: 스키마 0화 후 업로드 반영  
  - **로그는 생성하지 않음** (설계 의도)  
  - 파일 재업로드 시 또 증가하므로 주의  
  - 일부 동의어/오타 자동 보정

## 데이터 스키마
- **장소(3고정)**: `동아리방`, `비행장`, `교수님방`
- **계층 구조**: `장소 → 상위 → 하위 → (최하위) → 품목배열` (리프는 항상 배열)
- **카테고리 예시**
  - `공구`: `[수리, 납땜 용품, 드라이버, 그외 공구]`
  - `소모품`: 일부 3단 (`펜타 가드 → {새거, 중고, 기타}`, `테이프 → {필라멘트, 양면, 종이&마스킹, 절연, 그외 테이프}`)
  - `드론 제어부`: `[FC, ESC, 모터, …]`
  - `조종기 개수` / `기체 개수`: `[학교, 개인]`
- **Firebase 키 제약**  
  - 빈 문자열 / 금지문자 (`. # $ / [ ]`) 포함 키는 저장 시 제거됨

## 빠른 시작
> 환경: Ubuntu / Node.js 18+ 권장

```bash
# 클론
git clone https://github.com/x2okmiin/dokkaebi-inventory.git
cd dokkaebi-inventory

# 의존성 설치
npm install

# 환경변수 작성
printf "REACT_APP_VERSION=0.0.0-dev\n" > .env.local

# 개발 서버
npm start
````

접속: `http://localhost:3000`
로그인: 학번 + 이름 + 비밀번호 `2500` (관리자 모드 전환)

## 테스트 체크리스트

1. 수정 패널 버튼(＋/－/✎/📝) 동작, 카테고리 접힘 없음
2. 팝업 열자마자 `details` 전부 펼쳐짐
3. 반응형 그리드 (2×2, 3열+전체 가운데)
4. 검색 클릭 시 해당 위치 이동 + 집계 정확
5. Excel 내보내기: 시트1(재고현황, 0→nan), 시트2(품목별 합계)
6. 로그 페이지: 필터/내보내기/자동 병합 정상
7. 일괄 추가(베타): 합산·초기화 모드 모두 확인, 로그 미생성
8. Firebase 저장: 키 살균 경고만, 오류 없음
9. 모바일: 카드 스크롤/로그아웃 버튼 표시
10. 자동 로그아웃: 10분 무활동 시 동작

## 장애 · 디버깅

* **Firebase set 실패**: 빈키/금지문자 존재 → 저장 직전 살균
* **ESLint no-useless-escape**: 정규식 `const FORBIDDEN_KEY_RE = /[.#$/[\]]/;` 유지
* **모바일 스크롤/버튼 미표시**: `.card-body{overflow:auto}` 확인
* **팝업 접힘**: `requestAnimationFrame`으로 열자마자 `details.open=true` 강제
* **스플래시 미닫힘**: RTDB 지연 시 8초 후 강제 닫힘

## FAQ

* **일괄 추가 로그 안 남음?** → 의도된 설계 (재고만 변경)
